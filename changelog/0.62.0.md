# pgmold-sqlparser 0.62.0 Changelog

Fork-only release. Covers fork-side changes since 0.61.0; no upstream sync.

**Breaking changes:**

- PostgreSQL: rename `Statement::Comment.relation` to `table_name` for consistency with sibling AST nodes (`AlterPolicy.table_name`, `CreatePolicy.table_name`, `DropPolicy.table_name`, `AlterTrigger.table_name`) [#31](https://github.com/fmguerreiro/datafusion-sqlparser-rs/pull/31)

**Other:**

- PostgreSQL: `COMMENT ON FUNCTION` / `PROCEDURE` / `AGGREGATE` now parse argmode (`IN`/`OUT`/`INOUT`/`VARIADIC`) and argname tokens in the argument list, matching server syntax [#28](https://github.com/fmguerreiro/datafusion-sqlparser-rs/pull/28)
- PostgreSQL: cleanup of `CommentObject`, `AlterType`, and `AlterDefaultPrivileges` Display arms and parser keyword dispatch (no behavior change) [#30](https://github.com/fmguerreiro/datafusion-sqlparser-rs/pull/30)
